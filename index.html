What it is

Discipline,

Claude Code is Anthropic's CLI for AI-assisted engineering. By default, the agent is helpful — to a fault. claude-config replaces "helpful" with "rigorous": a set of rules and reviewers that fire before the agent acts, not after the work breaks.

Five gates that bracket the work

Discipline as
infrastructure.

01 Define the problem Stops solving the wrong problem. No coding before the problem is named.
02 Systems analysis Stops "just a small change" surprises. 60-second scan of what the change touches.
03 Fat marker sketch Stops detailed design on the wrong shape. Sketch the structure first.
04 Goal-driven verify Stops "should work" claims. Every step names a pass/fail check.
05 PR validation Stops silent merges. Test plan runs before "ready" is allowed.

Override any gate — by naming the risk you're willing to take.

DEFINE.

Most agents solve. This one interrogates first.

"What's the actual problem you're solving?"

Before any code, plan, or design — the gate demands problem definition first. Who's affected, what evidence supports the framing, what's the cost of being wrong.

Skip phrases like "CTO approved" or "ship by Friday" don't bypass it — they trigger it. The only valid override names the specific risk you accept. That's what thought partnership looks like in code: refuse the shortcut, demand the framing, let you decide if the risk is worth it.

Solve the right problem first
Disagreement discipline

When you push back,
the agent holds.

No "you're right" without new evidence.

× Not new evidence

"Trust me" "I'm the senior here" "We already decided" "Just do it" "Stop pushing back"

✓ New evidence

New data / runtime measurements Code or files unread A new constraint A cited source (RFC, spec) A specific technical claim

Capitulating without evidence is sycophantic. Three legitimate shapes only: hold, reverse-with-citation, yield-while-preserving-judgment.

Same rules. Leadership work.

Every skill inherits the gates. No skill flatters you into a plan.

  • /onboardNew-leader ramp scaffold. Refuses to read raw 1:1 notes — confidentiality is enforced by the harness, not promised.
  • /strategy-doc90-day plan author. Layered challenge pass — completeness, quality, consistency. Won't ship a strategy nobody pushed back on.
  • /swotLandscape analysis. Cross-session memory accumulation with built-in challenge layer — separates observation from intervention.

  • /presentAggregate-framed reflect-back decks. Source-controlled markdown — your strategy isn't a slide PDF, it's a diffable artifact.
